Core Rules

Skill Tests

When success is uncertain, roll a pool of 2–5 twenty-sided dice (d20). Each die that rolls equal to or under your target number scores one success. Meet or beat the difficulty to pass.

  1. Choose attribute + skill: GM sets combination (e.g. STR + Melee Weapons). Target number = attribute + skill rank.
  2. Set difficulty: Normally 1–5 successes required.
  3. Roll dice pool: Start with 2d20; buy up to 3 more with Action Points before rolling.
  4. Count successes: Each d20 ≤ TN = 1 success. Natural 1 = critical success (2 successes). Tag skills: any die ≤ skill rank = critical success.
  5. Compare to difficulty: Pass if successes ≥ difficulty. Extra successes become Action Points.
  6. Complications: Each natural 20 causes a complication after resolution (or GM gains 1 AP).

Test Difficulty

DifficultyExample
0Routine; auto-success unless you choose to roll
1Close-range shot, simple lock
2Reinforced door, treat injury
3Identify poison, disarm trap
4Complex hack, landmine
5Convince enemy to stand down, long-range storm shot

Opposed Tests

One side rolls first; their successes become the other's difficulty. In combat opposed tests, the defender spends AP first if buying dice. Environmental modifiers add to the winner's success total instead of difficulty.

Assistance

Roll 1d20 (no AP buys); add successes to ally's pool if they scored at least 1 success. Group tests: one leader, others assist.

Action Points

  • Generated by extra successes on tests; group pool max 6 AP
  • Buy d20s: 1st die = 1 AP, 2nd = 2 AP, 3rd = 3 AP (max 5d20 total per test)
  • After test: Obtain information (1 AP), reduce time (1 AP), improve quality (1 AP), extra minor action (1 AP), extra major action (2 AP, +1 difficulty)
  • If no AP to buy dice, each AP you need gives the GM 1 AP (buy dice only)
  • GM starts each quest with 1 AP per player; no maximum

Luck

Luck points = Luck attribute at start of each quest. Refresh at milestones or new quest.
